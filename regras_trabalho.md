# Critérios de Avaliação — Trabalho Prático

A tabela abaixo apresenta os critérios de avaliação do trabalho prático:

## Requisitos da Aplicação

| Critério | Pontuação |
|---|---|
| A aplicação é acessível apenas após autenticação | 0,5 |
| Cadastro de novos usuários | 0,5 |
| Quando autenticado, o usuário deve poder atualizar seu cadastro | 0,5 |
| Deve existir alguma base de dados no back-end | 0,5 |
| Deve ser possível compartilhar dados entre usuários | 0,75 |
| O front-end deve ser responsivo (desktop e mobile) | 1,0 |
| A aplicação deve ficar disponível 24/7 em algum servidor | 0,75 |
| **Total (requisitos da aplicação)** | **4,5** |

## Outros Aspectos Avaliados sobre o Código

| Critério | Pontuação |
|---|---|
| Organização da entrega e organização do código | 1,0 |
| Documentação do código | 0,5 |
| **Total (outros aspectos)** | **1,5** |

## Apresentação

| Critério | Pontuação |
|---|---|
| Apresentação do ponto de vista do usuário (funcionalidades da aplicação) | 1,0 |
| Apresentação do front-end | 0,75 |
| Apresentação do back-end | 0,75 |
| Respostas às perguntas | 0,5 |
| **Total (apresentação)** | **3,0** |

## Participação na Apresentação (fator multiplicativo)

| Critério | Valores possíveis |
|---|---|
| Participação na apresentação | 0 ou 1 |

O fator multiplicativo será **0 ou 1**. Se você participar da apresentação, falando seu nome e explicando qual foi a sua contribuição no projeto, o fator multiplicativo será **1**. Caso contrário, será **0**.

## Presença nas Apresentações dos Demais Grupos

| Critério | Pontuação |
|---|---|
| Presença nas apresentações dos demais grupos | 1,0 |

> **Nota:** Há também a previsão de **0,5 ponto extra** para quem fizer perguntas (bem fundamentadas) para as demais equipes.

## Resumo da Pontuação

| Item | Pontos |
|---|---|
| Requisitos da aplicação | 4,5 |
| Outros aspectos | 1,5 |
| Apresentação | 3,0 |
| Presença nas apresentações dos demais grupos | 1,0 |
| **Total** | **9,0** |

## Fórmulas de Cálculo

**Nota da entrega e apresentação:**
```
Nota = (requisitos da aplicação) + (outros aspectos) + (participação na apresentação) × (apresentação) + (presença nas apresentações dos demais grupos)
```

**Nota do projeto web:**
```
Nota do projeto web = min(10, Nota da entrega e apresentação)
```

## Instruções de Entrega

Utilize esta tarefa para realizar a **entrega final do Trabalho Prático**. É suficiente que apenas **um membro de cada grupo** faça a entrega.

Um único **arquivo compactado (`.zip`)** deve ser entregue, contendo:

- Um diretório **`src/`** com todo o código-fonte da aplicação (front-end, back-end e outros recursos de código necessários);
- Um arquivo **`README.md`**, contendo:
  - Uma breve descrição do trabalho (o que vocês fizeram);
  - Os nomes e matrículas dos membros do grupo;
  - Um link (URL) para o repositório do projeto (deverá ser público ou estar compartilhado com o usuário do professor na referida plataforma);
  - Um link (URL) para acesso à aplicação Web desenvolvida e disponível 24/7 (até que a nota seja atribuída ao trabalho).

> **Importante:** No diretório de código-fonte, **não adicionem o diretório `node_modules`**, pois as dependências podem ser recuperadas utilizando o arquivo `package.json`.
